<p>Vous avez envie de contribuer à l'Open Source, et c'est tout à votre honneur. Alors vous vous dites &quot;<em>Allez, ce week-end, je plie ma première Pull Request</em>&quot;. Et là, c'est le drame. Vous arrivez sur GitHub, vous tapez trois mots-clés, et vous vous retrouvez noyé sous des milliers d'issues datant de 2014, des projets abandonnés ou des tâches qui demandent un doctorat en physique quantique.</p>
<p>Décourageant un peu non ?</p>
<p>Parce qu'on connaît tous cette sensation de tourner en rond en cherchant le fameux ticket &quot;Good First Issue&quot; qui ne soit pas juste &quot;corriger une faute de frappe dans le README&quot;, j'ai décidé aujourd'hui, de vous aider ! Grâce notamment à une véritable boussole dans cette jungle open source, nommée <strong>
<a href="https://k-dash.github.io/contrib-fyi/">Contrib.FYI</a>
</strong>.</p>
<img src="https://korben.info/cdn-cgi/image/width=1200,fit=scale-down,quality=90,f=avif/contrib-fyi-open-source/contrib-fyi-open-source-2.png" alt="" loading="lazy" decoding="async">
<p><em>L'interface épurée pour trouver votre prochaine mission (
<a href="https://k-dash.github.io/contrib-fyi/">Source</a>
)</em></p>
<p>Ce site c'est un moteur de recherche ultra-minimaliste qui agrège les tâches étiquetées &quot;Help Wanted&quot; et &quot;Good First Issue&quot; à travers tout l'écosystème GitHub.</p>
<p>Développé par un certain <strong>k-dash</strong>, ça se présente comme un tableau de bord où vous pouvez filtrer les opportunités, et c'est beaucoup plus agréable que la recherche native de GitHub qui est aussi accueillante qu'une administration française un lundi matin.</p>
<p>Vous avez même une fonctionnalité &quot;My Picks&quot; pour mettre de côté les tickets qui vous tapent dans l’œil, histoire de ne pas les perdre si vous devez aller promener le chien entre temps. Une autre feature sympa, c'est l'historique &quot;<em>Recently Viewed</em>&quot; qui permet de retrouver une issue après qu'on ait fermé un onglet par erreur.</p>
<p>
<img src="https://korben.info/cdn-cgi/image/width=1200,fit=scale-down,quality=90,f=avif/contrib-fyi-open-source/contrib-fyi-open-source-3.png" alt="" loading="lazy" decoding="async">
</p>
<p>Alors oui, des agrégateurs comme ça, il en existe d'autres. Je pense évidemment à
<a href="https://goodfirstissue.dev/">GoodFirstIssue.dev</a>
ou
<a href="https://www.codetriage.com/">CodeTriage</a>
mais <strong>Contrib.FYI</strong> a ce petit côté outil fait par un dev pour des devs qui ne cherche pas à vous vendre une formation ou à vous faire rejoindre une newsletter.</p>
<p>Du coup, maintenant vous n'avez plus d'excuse pour ne pas mettre les mains dans le cambouis. Que vous soyez un vétéran du code qui veut juste dépanner sur du Rust ou un débutant qui veut comprendre comment fonctionne une PR, y'a de quoi faire !</p>
<p>Et si vous cherchez d'autres outils pour pimper votre expérience GitHub, je vous avais déjà parlé de
<a href="https://korben.info/git-who-outil-revolutionne-analyse-contributions-git.html">Git-Who</a>
pour analyser vos contributions ou encore de
<a href="https://korben.info/github2file-simplifie-partage-code-chatbots.html">GitHub2File</a>
pour donner du contexte à vos IA préférées. Allez jeter un oeil, ça vous plaira peut-être !</p>
